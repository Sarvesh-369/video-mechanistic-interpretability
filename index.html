Qwen3-VL Analysis

Perception stability vs. counting limits: a visual study of internal representation layers.

Motivation & Linear Probing
The Core Question: Do counting failures originate in perception (vision encoder failing to resolve state transitions) or in reasoning (decoder failing to aggregate events in context)?
Feature Extraction (Layer -2): During a forward pass of a video, we register a PyTorch forward hook on the text decoder self-attention modules at the second-to-last transformer layer (layer -2) to capture raw token activations.
Spatial Pooling & Alignment: Visual tokens are represented as a 3D grid (T_out, H_out, W_out, 4096). We average the 4096-dimensional vectors spatially across the spatial coordinates (H, W) to produce a single (T_out, 4096) feature trajectory—yielding one vector per second.
Classifier Training (Count ≤ 3): We train a Logistic Regression probe on the pooled 4096-dimensional representation vectors from low-count videos (≤ 3 events) to learn a linear boundary mapping representations to the binary ground-truth state (ON/OFF, Wall A/B, or Cool/Warm). We perform the prediction frame-by-frame, where each individual frame's representation vector is evaluated independently.
Perceptual Evaluation (Count ≥ 5): We evaluate the trained probe on representations from high-count trap videos (≥ 5 events) frame-by-frame. High test accuracy indicates that the model's representations preserve the visual states frame-by-frame, verifying that the bottleneck is not perceptual.

Motivation & Logit Lens Process
The Core Question: If the intermediate representations successfully track the object state, why does the model output incorrect counts? We use Logit Lens to project layer-wise representations directly into the vocabulary token space to observe count logit emergence.
Target Token: Project the hidden state hl of the final query token at each decoder layer l in [0, 36].
Vocabulary Mapping & Dimension Matching: We map the 4096-dimensional intermediate hidden state back to the vocabulary space without training new layers. Instead, we use the pre-trained weights of the model's own language model head (lm_head), which acts as a linear layer of shape (4096, 151936) that projects the 4096-dimensional hidden vector to logits over the vocabulary tokens.
Probability Distribution: Apply Softmax over target digit tokens (e.g. '1', '2', '3', '4', '5') to map the token probability emergence at each layer.
